1. 브랜치 전략 및 작업 기간
브랜치 생성: main 브랜치에서 분기하여 feature/[본인 깃 아이디] 형식으로 브랜치 이름을 생성해 주세요.
작업 기간: 해당 작업은 1주일 동안 진행됩니다.

2. 주요 작업 내용 (QA 및 API 점검)
QA 이슈 해결: 각자 배정된 QA 이슈들을 해결해야 합니다.

API 정상화: 배포 과정에서 발생했을 수 있는 오류를 확인하기 위해, 전체적인 API 연결 상태를 체크하고 정상화하는 작업을 수행해 주세요.

응답 데이터 전달: 각 API 요청에 대한 답(특히 매거진 본문 등)을 전달해야 합니다.

3. 데이터 관련 작업
더미 데이터 생성: 각자 계정을 하나씩 생성하여 더미 데이터를 만들어 주세요.

데이터 입력: 생성한 더미 데이터는 배포된 서버 내의 DB에 직접 넣어주시고, 공용으로 사용할 아이디에도 데이터를 보충해 주세요.

4. 소통 및 기록
의논 및 문의: 개발 중 논의가 필요한 사항은 언제든 카카오톡을 통해 공유해 주세요.

트러블슈팅 기록: 작업 중 발생하는 문제나 해결 과정은 노션(Notion)의 트러블슈팅 페이지에 기록하는 것이 권장됩니다.
